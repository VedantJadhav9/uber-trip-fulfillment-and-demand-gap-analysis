# Uber Supply-Demand Gap Analysis

An end-to-end data analytics project focused on identifying supply-demand mismatches in Uber ride requests using Python, SQL, and Excel dashboards.

This project analyzes ride request patterns, cancellations, driver availability, and trip fulfillment trends to generate actionable business insights for improving operational efficiency and customer experience.

---

## Project Overview

Uber frequently faces ride fulfillment issues due to:
- Driver cancellations
- No cars available
- Peak-hour demand surges

This project aims to identify:
- When supply-demand gaps occur
- Which locations are most affected
- Why trips remain unfulfilled
- How Uber can improve fulfillment rates

## Key Insights

### 1. Trip Fulfillment Overview
- Only **56.5%** of ride requests were successfully completed
- **43.5%** remained unfulfilled

### 2. Time Slot Analysis
- Evening (5 PM–9 PM) had highest demand
- Late Night showed maximum "No Cars Available"
- Early Morning had highest cancellation rates

### 3. Pickup Point Analysis
- Airport trips had higher unfulfilled percentages
- City pickups showed better completion rates

### 4. Hourly Demand Patterns
Two major demand peaks:
- Morning Peak: 8–10 AM
- Evening Peak: 5–9 PM

### 5. Service Gap Findings
- Largest gaps observed at Airport during Late Night
- Driver shortages significantly impacted ride fulfillment
